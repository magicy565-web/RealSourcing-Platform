## 1. 核心洞察：Sourcing 的本质是“建立信任”

您完全正确。经过对买家（Dropshipping/DTC 品牌主）和工厂（国内制造商）两侧需求的深度研究，我们发现，**Sourcing 的核心挑战并非信息发现，而是信任建立**。阿里巴巴和 Global Sources 解决了信息不对称，但留下了更难的信任鸿沟。买家害怕遇到“中间商”、“小作坊”，担心产品质量、交期、MOQ（最小起订量）等一系列问题。

因此，RealSourcing 的核心价值主张不应仅仅是“连接”，而应是“**提供建立信任的工具和环境**”。Webinar 和 1:1 会议正是实现这一点的绝佳载体，因为它们比静态的商品页面和冰冷的聊天消息更能传递真实感和专业度。

## 2. 双边需求全景地图

### 买家侧 (DTC/Dropshipping) 需求：从“发现”到“下单”的全链路

| 阶段 | 核心需求 | 痛点 | RealSourcing 解决方案 |
|---|---|---|---|
| **1. 发现 (Discovery)** | 找到潜在供应商 | 信息过载，难辨真伪；害怕中间商 | **工厂认证体系**；**Webinar 直播广场**（看真人、看车间） |
| **2. 筛选 (Vetting)** | 评估供应商能力 | 资质难验证；沟通效率低；样品周期长 | **工厂主页（含认证、评级）**；**1:1 视频会议**（实时看厂、看样品） |
| **3. 拿样 (Sampling)** | 验证产品质量 | 样品费用高；寄送时间长；样品与大货可能不符 | **样品单在线交易**；**平台验货报告** |
| **4. 议价 (Negotiation)** | 敲定价格、MOQ、交期 | 文化差异；信息不对称；小订单议价能力弱 | **RFQ（询价）功能**；**平台集采/拼单模式** |
| **5. 下单 (Ordering)** | 签订合同，支付定金 | 支付安全风险；合同约束力弱 | **平台担保交易（类似 Trade Assurance）**；**标准化合同模板** |
| **6. 跟单 (Production)** | 跟踪生产进度 | 进度不透明；延期风险 | **工厂生产进度更新**；**平台中期验货服务** |
| **7. 尾款与物流 (Fulfillment)** | 支付尾款，安排发货 | 尾款支付风险；物流选择复杂 | **平台担保支付**；**集成第三方物流服务** |

### 工厂侧 (国内制造商) 需求：从“曝光”到“收款”的全链路

| 阶段 | 核心需求 | 痛点 | RealSourcing 解决方案 |
|---|---|---|---|
| **1. 曝光 (Exposure)** | 获取海外买家流量 | 线下展会成本高；线上广告投放难；B2B 平台询盘质量低 | **公域 Webinar 直播广场**；**平台内容推荐（工厂故事）** |
| **2. 转化 (Conversion)** | 将流量转化为询盘 | 网站体验差；信任感建立难；无法展示动态实力 | **企业级工厂主页**；**自助发起 Webinar**；**1:1 会议预约系统** |
| **3. 对接 (Engagement)** | 高效响应买家需求 | 跨时区沟通；语言障碍；反复寄样 | **1:1 视频会议**；**AI 实时翻译**；**在线样品展示** |
| **4. 报价 (Quotation)** | 管理报价和订单 | 报价流程繁琐；订单管理混乱 | **标准化 RFQ 响应系统**；**订单管理仪表盘** |
| **5. 签约收款 (Contract)** | 确保交易安全 | 担心买家跑单；收款周期长 | **平台担保交易**；**快速收款通道** |

## 3. MVP 重新定义：构建最小化“信任闭环”

基于以上需求地图，MVP 的目标应是**让买卖双方能完成一次最小化的、基于信任的交易互动**。这个闭环的核心不是“自助”，而是“透明”和“高效”。

**MVP 核心用户故事：**

1.  **工厂完善“信任档案”**：工厂入驻后，可以在**工厂仪表盘**中上传营业执照、认证证书（如 ISO9001）、车间照片/视频、主打产品等，构建一个可信的“工厂主页”。
2.  **工厂发起“线上开放日” (Webinar)**：工厂可以在仪表盘中**自助发起一场 Webinar**，主题可以是“新品发布”、“车间实况”、“工艺讲解”等，并设定开播时间。
3.  **买家通过“直播”建立初步信任**：买家在平台看到工厂发起的 Webinar，通过观看直播（看真人、看环境、看产品），对工厂建立初步的、感性的信任。
4.  **买家发起“深度验厂” (1:1 会议)**：在 Webinar 直播间或工厂主页，买家可以点击“**预约 1:1 视频会议**”按钮，进入一个带日历的预约页面，向工厂发起一个“深度验厂”或“样品详谈”的会议邀约。
5.  **双方通过“视频会议”建立深度信任**：在 1:1 视频会议中，工厂可以实时展示样品细节、回答买家问题，双方建立更深度的信任关系。
6.  **买家发起“样品订单”**：会议结束后，买家可以在平台上向工厂**发起一个样品采购订单**，并通过平台完成支付。

### MVP 功能边界 (MoSCoW)

| 功能 | 类别 (MoSCoW) | 理由 |
|---|---|---|
| **工厂仪表盘 (含资料/产品/Webinar管理)** | **Must Have** | 信任的基础，供给侧自运营的起点 |
| **自助创建/发起 Webinar 功能** | **Must Have** | 建立初步信任的核心载体 |
| **1:1 会议自助预约系统** | **Must Have** | 建立深度信任的核心工具 |
| **样品单在线交易功能** | **Must Have** | 完成最小商业闭环，验证支付通路 |
| 真实视频流 (Agora/WebRTC) | **Should Have** | 对建立信任至关重要，但初期可用占位符，引导线下进行 |
| AI Meeting Reel | **Could Have** | 属于效率优化，可在 MVP 成功后快速跟进 |
| 平台担保交易 (Trade Assurance) | **Won't Have** | 流程复杂，MVP 阶段可先从样品单小额交易开始 |
| RFQ / 拼单 / 物流集成 | **Won't Have** | 属于规模化阶段的需求 |

## 4. 下一步行动计划：信任闭环 MVP 开发

**目标**：用 3-4 周时间，完成能让买卖双方跑通“了解→信任→样品交易”核心闭环的 MVP。

### P0: 供给侧信任基建 (立即开始)

1.  **重构 `FactoryDashboard.tsx`**
    - **任务**: 增加“工厂资料管理”（上传认证）、“我的产品管理”（增删改查）、“我的 Webinar 管理”（创建/编辑/查看）三大核心模块。
    - **依赖**: 后端需提供 `factories.updateProfile`、`products.create/update/delete`、`webinars.create/update` 等路由。

2.  **开发“创建/编辑 Webinar”页面**
    - **任务**: 创建一个独立的表单页面，让工厂可以自助创建 Webinar。

### P1: 需求侧连接与交易

1.  **开发 `/meeting/new` 会议预约页面**
    - **任务**: 创建带日历组件的会议预约页面。

2.  **开发“样品订单”功能**
    - **任务**: 在产品页面或 1:1 会议后，允许买家创建一个样品订单，包含产品、数量、收货地址，并生成一个待支付链接（初期可集成 Stripe/PayPal）。
    - **依赖**: 后端需提供 `orders.createSampleOrder` 路由和支付集成。

这个方案比之前的更完整，它不仅解决了“自运转”的技术卡点，更重要的是，它**围绕“建立信任”这一核心商业价值来设计产品闭环**，使平台对买卖双方都更具吸引力。
